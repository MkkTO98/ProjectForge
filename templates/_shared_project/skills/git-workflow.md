# Skill: Git Workflow

## Default MacroForge-style policy
- Inspect before modifying: `git status`, `git diff`.
- Commit small coherent units.
- Commit messages must describe why, not only what.
- Pushing requires approval unless project policy explicitly permits it.
- Branches are introduced when concurrent workstreams or risk require them.

## Branch clarification
A branch is an independent line of development. A single-branch workflow is simpler. Feature branches are safer when multiple tasks progress in parallel or when agents make larger changes.
