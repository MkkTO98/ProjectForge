# Contradictions

Contradiction records capture architectural disagreements without forcing a winner.

Use them when patterns solve overlapping problems with different assumptions, costs, and failure modes. Each record should explain where each approach wins, where it loses, what evidence exists, and how ProjectForge constraints affect the choice.

Examples:

- event sourcing vs snapshot persistence
- graph orchestration vs linear task workflows
- autonomous multi-agent delegation vs single-agent tool use
- vector memory vs file-backed indexes
